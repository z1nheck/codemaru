# Codemaru v0.2 — Nano + 処理側の工夫(検証用)

Chrome内蔵の **Gemini Nano(Prompt API)** を主エンジンに、英→日翻訳する拡張。
コードは原文のまま温存し、Nanoへの指示で**日本語の語順を自然に組み直す**のが狙い。
通信ゼロ・無料・キー不要。Nano非対応端末では内蔵MT(Translator API)に自動フォールバック。

## v0.2で入れた工夫

- **few-shot**:コードを含む英文→理想の日本語の訳例をsystem promptに同梱(語順の型を学習)。
- **コードを名詞として温存**:`<code>`/`<pre>` や URL・`--flag`・パス・`v1.2.3`・`ENV_VAR` を
  `⟦N│literal⟧` トークンに退避。「訳すな・名詞として自然な位置に置け」とNanoに指示。
- **タグ検証+自動リトライ**:訳文にトークンが全部戻ってるか機械チェック。欠けたら1回だけ訳し直し。
  それでも欠けたコードは末尾に補完して、コードを絶対に消さない。
- **文単位+前文の文脈**:断片で投げず、前の文を「訳さない文脈」として渡し指示語・語順を安定化。
- **軽い用語集**:function=関数 等の頻出語を固定+API/JSON等は英語のまま。
- **翻訳メモリ**:同じ文・用語は同じ訳を再利用(一貫性とスピード)。
- **エンジンinterface化**:`Engines.nano` / `Engines.mt` の2実装。将来 `Engines.cloud`(自前キーBYOK)
  を足すだけで差し替え可能(中身は未実装。継ぎ目だけ用意)。

## 入れ方(初回1回だけ)

1. このフォルダを解凍。
2. `chrome://extensions` →「デベロッパー モード」ON。
3. 「パッケージ化されていない拡張機能を読み込む」→ `codemaru` フォルダを選択。
4. アイコンをピン留め。

## 使い方 / 比較

1. 英語の技術ドキュメントを開く(普通のhttpsページ)。
2. アイコン →「このページを翻訳」。段落の下に日本語訳が入る。
3. ポップアップ下部に `engine: Gemini Nano` などと使ったエンジンが出る。
4. **比較**:
   - 「高速モード(内蔵MT)で訳す」ON → 純粋MT版。Nano版と**語順**を見比べる。
   - 「コードも訳す」ON → コード退避を切り、Google翻訳相当の壊れ方を再現。
5. 「原文に戻す」で全消し。

見てほしいのは(1)コードが文中で名詞として自然な位置に来てるか、(2)同じ語の訳がブレてないか、
(3)MT版と比べて語順がどれだけマシか、の3点。

## うまく動かないとき

- 「内蔵AIが見つかりません」→ Chromeを最新に更新。必要なら `chrome://flags` で
  Prompt API for Gemini Nano / Translation API を Enabled にして再起動。
- **初回のNanoモデルDL**:約2GBがバックグラウンドで落ちます。`chrome://on-device-internals`
  で進捗確認可。落ち切る前は availability が downloadable/downloading になり、
  その場合は一度ページ上をクリック→もう一度「翻訳」を押すと確実です。
- Nanoはデスクトップ専用(モバイル/Firefox不可)。非対応端末では自動でMTに切替。
- `chrome://`・拡張ストア・PDFビューア等では動きません。

## 既知の限界(次の候補)

- リンク/太字などインライン装飾はプレーン化(コードのみ位置保持)。
- Nanoは小型モデルゆえ、長文・専門語で訳の揺れが出ることがある。
  → 物足りなければ「draft→naturalize の二段階」や「苦手チャンクだけBYOLで高品質モード」を後付け予定。
- ソース言語は手動選択(自動判定は未実装)。
